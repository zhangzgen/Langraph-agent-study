- **规范化文档注释**：所有新增函数必须包含专业的中文文档字符串（Docstring，如 `""" """` 格式）。注释内容必须结构化，明确包含以下三项：
  - **功能说明**（Description）：简明扼要地描述该函数的核心作用。
  - **参数说明**（Args）：详细列出每个参数的名称、类型及用途。
  - **输出说明**（Returns）：明确说明返回值的类型及其代表的含义。

- **版本控制授权拦截**：在未经用户明确指令授权的情况下，严禁 Agent 擅自执行任何 Git 代码提交（Commit）或远程推送（Push）操作。

- **克制化封装与核心逻辑优先**：
  - **避免过度抽象**：对于缺乏复用场景且代码行数较少的代码片段，禁止强行剥离为独立函数，保持代码的内聚与直观。
  - **聚焦主干逻辑**：代码编写应以简洁和高可读性为原则，聚焦于主干业务逻辑的实现，无需过度设计或冗余编写边缘异常的兜底（Fallback）处理机制。
